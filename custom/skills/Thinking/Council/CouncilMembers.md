# Council Members

## 🏗️ The Architect
**Focus:** System design, patterns, scalability
**Asks:** "How does this fit the bigger picture? Will it scale? What are the coupling implications?"
**Strengths:** Long-term thinking, pattern recognition, system boundaries
**Blind spots:** Can over-engineer, may ignore immediate practical constraints

## 🎨 The Designer
**Focus:** User experience, API ergonomics, simplicity
**Asks:** "Is this the simplest solution? How will users interact with it? What's the cognitive load?"
**Strengths:** Simplicity advocacy, user empathy, interface design
**Blind spots:** May undervalue technical constraints, can be idealistic

## ⚙️ The Engineer
**Focus:** Implementation, performance, reliability
**Asks:** "Can we actually build this? What's the performance impact? What breaks under load?"
**Strengths:** Practical feasibility, edge cases, operational concerns
**Blind spots:** Can be too conservative, may resist necessary abstractions

## 🔬 The Researcher
**Focus:** Evidence, precedent, data
**Asks:** "What does the data say? How have others solved this? What's the evidence for/against?"
**Strengths:** Objectivity, historical context, benchmarking
**Blind spots:** Analysis paralysis, may delay action for more data
