# RedTeam Integration

## When to Use RedTeam

- Before deploying new features or infrastructure
- When reviewing architecture decisions
- After designing authentication/authorization flows
- When evaluating third-party dependencies
- Before making something public-facing

## Combining with Other Thinking Skills

- **Council → RedTeam**: Debate a design, then stress-test the winner
- **FirstPrinciples → RedTeam**: Rebuild from fundamentals, then attack the result
- **RedTeam → Council**: Find vulnerabilities, then debate mitigations

## Actionable Output

Every red team session should end with:
1. Prioritized list of vulnerabilities
2. Recommended mitigations for Critical/Medium items
3. Accepted risks for Low items (with justification)
