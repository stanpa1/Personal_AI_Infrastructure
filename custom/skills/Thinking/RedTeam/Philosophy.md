# RedTeam Philosophy

## Core Principles

1. **Assume breach** — Don't ask "can this be attacked?" Ask "how would it be attacked?"
2. **Think like the adversary** — What's the laziest, cheapest, most effective attack?
3. **Chain weaknesses** — Individual low-severity issues can combine into critical exploits
4. **Question assumptions** — "This is safe because..." is where vulnerabilities hide
5. **Time dimension** — Attacks evolve; what's safe today may not be tomorrow

## Severity Classification

- 🔴 **Critical** — Data loss, security breach, system down, financial impact
- 🟡 **Medium** — Degraded service, partial failure, workaround exists
- 🟢 **Low** — Minor inconvenience, cosmetic, theoretical risk

## The Attacker's Advantage

Defenders must protect everything. Attackers only need one way in.
Focus on the weakest links, not the strongest defenses.
