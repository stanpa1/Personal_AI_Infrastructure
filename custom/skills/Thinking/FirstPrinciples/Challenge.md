# Challenge Workflow

Question every assumption using the "5 Whys" technique.

## Steps

1. **List assumptions** from Deconstruct phase (or gather fresh)
2. **5 Whys** for each assumption — dig to the root
3. **Classify** — Proven (keep) vs. Inherited (question) vs. False (discard)
4. **Impact analysis** — What changes if we discard each false assumption?

## Output Format

```
# 🔬 First Principles: Challenge Phase

## Assumption: [assumption text]
1. Why? → [answer]
2. Why? → [answer]
3. Why? → [answer]
4. Why? → [answer]
5. Why? → [root cause]
**Verdict:** ✅ Proven / ❓ Inherited / ❌ False
**If false:** [what changes]

[Repeat for each assumption]

## Summary
- **Keep:** [proven assumptions]
- **Question:** [inherited — need more evidence]
- **Discard:** [false — opens new possibilities]
```
