# FirstPrinciples — Deconstruct → Challenge → Reconstruct

Strip away assumptions to find fundamental truths, then rebuild solutions from the ground up.

## The Framework

### Phase 1: Deconstruct
Break the problem into its smallest components. What are the atomic facts?

### Phase 2: Challenge
Question every assumption. "Why?" five times. Which beliefs are inherited vs. proven?

### Phase 3: Reconstruct
Build back up from proven fundamentals only. What solution emerges?

## Workflows

- **Deconstruct** (`Deconstruct.md`): Phase 1 only — break down the problem
- **Challenge** (`Challenge.md`): Phase 2 only — question assumptions
- **Reconstruct** (`Reconstruct.md`): Full cycle through all 3 phases

## When to Use

- "We've always done it this way" situations
- Complex problems with many layers of abstraction
- When existing solutions feel overly complex
- Evaluating whether to build vs. buy
- Questioning industry "best practices"
