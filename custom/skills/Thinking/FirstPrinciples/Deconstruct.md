# Deconstruct Workflow

Break down a problem to its atomic components.

## Steps

1. **State the problem** — What are we trying to solve?
2. **List components** — What are all the pieces involved?
3. **Identify assumptions** — What are we taking for granted?
4. **Find fundamentals** — What is provably, physically, logically true?
5. **Map dependencies** — Which components depend on which?

## Output Format

```
# 🔬 First Principles: Deconstruction of [Topic]

## Problem Statement
[Clear statement of what we're solving]

## Components
1. [Component A]
2. [Component B]
...

## Assumptions (to be challenged)
- ❓ [assumption 1]
- ❓ [assumption 2]
...

## Fundamental Truths
- ✅ [proven fact 1]
- ✅ [proven fact 2]
...

## Dependency Map
[Component A] → depends on → [Component B]
...
```
